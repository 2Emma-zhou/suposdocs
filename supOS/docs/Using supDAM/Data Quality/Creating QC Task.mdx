Create quality control tasks to check the quality of selected fields of QC rules.
1. Log in to supOS, and then select **Data Intelligence** > **Data Governance** > **Data quality** > **Quality control task**.
2. Click **New**, and then set the task information.

<ol>
<ol>
<li>Click <b>Add rule</b> under <b>Rule</b>, and then select multiple rules for the task to monitor.

:::info
For multiple rules, you can set the weight of each rule to differentiate corresponding priority.
:::

</li>
<li>Under <b>Execution cycle</b>, select execution method, and then set the triggering schedule for <b>Timed execution</b>.

:::info
- For **Manual execution**, you can directly click **Execute** under **Operation** after finishing task creation to execute it.
- For **Timed execution**, the task will be triggered at the scheduled time, and by clicking **Execute** under **Operation** after finishing task creation.
:::

</li>
<li>Under <b>Alarm settings</b>, set <b>Alarm mode</b>, <b>Alarm object</b> and <b>Alarm level</b>.</li>
</ol>
</ol>

![](https://wordpressfreezonex.oss-accelerate.aliyuncs.com/supDAM/29.png)

3. Click **Confirm**.
