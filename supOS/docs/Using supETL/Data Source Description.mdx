:::info
When adding HBase as data source, the following configurations on the X-ETL backend are needed.
:::

1. Add HBase cluster deployment network configurations to **dam-resource**, **flink-jobmanager-batch** and **flink-taskmanager-batch**.
```
kubectl edit deploy dam-resource
```
2. Press **i** to start editing, and add the following content between **spec** and **containers**.
```
hostAliases:
- ip: 192.168.x.x //hadoop cluster IP
hostnames:
- "ubuntu-hadoop01" //hadoop cluster hostname
- ip: 192.168.x.x
hostnames:
- "ubuntu-hadoop02"
- ip: 192.168.x.x
hostnames:
- "ubuntu-hadoop03"
- ip: 192.168.x.x
hostnames:
- "ubuntu-hadoop04"
```
![](https://wordpressfreezonex.oss-accelerate.aliyuncs.com/supETL/50.png)

3. Finish editing and do the same for **flink-jobmanager-batch** and **flink-taskmanager-batch**.
4. Check the container ID, and then access the container.
```
kubectl get po
kubectl exec -it dam-resource-5b558bfdd7-kxgv2 bash
```
![](https://wordpressfreezonex.oss-accelerate.aliyuncs.com/supETL/51.png)

5. Inside the container, execute the following content to check whether the mapping is added in **hosts** file.
```
cat /etc/hosts
```
6. Exit the container.
