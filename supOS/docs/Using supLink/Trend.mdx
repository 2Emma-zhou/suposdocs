### Configuring on web

1. Log in to supLink web, and then click **App Manage**.
2. Under **Base App**, click **Edit** on **Trend**.
![](https://wordpressfreezonex.oss-accelerate.aliyuncs.com/supLink/20.png)

<table>
    <thead>
      <tr>
        <th>Parameter</th>
        <th>Description</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td>Default horizontal</td>
        <td>Select whether to display trend chart horizontally on mobile by default.</td>
      </tr>
      <tr>
        <td>QR code</td>
        <td>Use supLink app to scan the code to access the application directly.</td>
      </tr>
      <tr>
        <td>Visible range</td>
        <td>Set the application visible range. Select roles to grant permissions when set <b>Visible range</b> to <b>Part visible</b> or <b>Part invisible</b>.</td>
      </tr>
      <tr>
        <td>Select group</td>
        <td>Select an app group for the application.</td>
      </tr>
      <tr>
        <td>Default trend</td>
        <td>Add trend to be default trend that shows on mobile.</td>
      </tr>
    </tbody>
</table>

3. Click **Add trend**, and then select the trend, click **Setting**.
4. Select attributes to display their value trend.
![](https://wordpressfreezonex.oss-accelerate.aliyuncs.com/supLink/21.png)
:::info
Trend set by admin will show on mobile supLink apps that do not have custom trend list configured.
:::
5. Click **Save**.

### Viewing Effect on Mobile

1. Install supLink mobile on your phone, start it and enter the supLink server address, tap **Start**.
2. Log in with the same account as supOS.
3. Under **Base App**, tap **Trend**, and you can see the trend list you added on web.
:::info
The **Base App** group name is set on web in **Group Manage**.
:::
4. Select a trend.
