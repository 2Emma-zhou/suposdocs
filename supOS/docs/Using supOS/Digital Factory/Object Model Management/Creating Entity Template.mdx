## Creating Entity Template

1. Log in to supOS, and then click <img src="https://wordpressfreezonex.oss-accelerate.aliyuncs.com/factoryModeling/7.png" /> to go to the design center.
2. Select **Object Model Management** > **Object Template** > **Entity Template**.
3. Click <img src="https://wordpressfreezonex.oss-accelerate.aliyuncs.com/factoryModeling/26.png"/> next to **Entity Template**, and then enter the basic information of the template.

</td>
</tr>
<tr>
<td>Compression Parameters</td>
<td>Compress data with SDT (Spining Door Transformation) to reduce the space that history records take up.

- <strong>Compression Value</strong>: Set a percentage of the value range, or an absolute value as the input parameter of SDT.
- <strong>Maximum Upload Cycle</strong>: The maximum time interval between 2 collecting records. When reaching the set value, and data has no changes, the last recorded value will be uploaded.

:::info
- The higher the <strong>Compression Parameters</strong>, the heavier the data is compressed.
- No matter how high the **Compression Parameters** is, the time interval of collecting data cannot exceed **Maximum Upload Cycle**.
:::

</td>
</tr>
</tbody>
</table>

</li>
<li>Under <strong>Alarm rules</strong>, click <strong>New</strong> to add an alarm that will be triggered when the temperature is higher than 85°C.

- Set **Condition** to **>** and <strong>Limit</strong> to 85.
- Set a buffer temperature of 5 by selecting **Value** to be **Dead Zone Type** and entering **5** under **Dead Zone Value**. An alarm will go off when the temperature hits 80°C.
