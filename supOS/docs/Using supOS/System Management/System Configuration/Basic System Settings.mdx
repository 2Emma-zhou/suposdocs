Manage basic system configurations. 

## Designing Theme
1. Log in to supOS, and then click <img src="https://wordpressfreezonex.oss-accelerate.aliyuncs.com/platform/7.png"/> at the upper-right corner to enter the design center.
2. Select <strong>System Configuration</strong> > <strong>System Settings</strong> > <b>Theme</b>.
<img src="https://wordpressfreezonex.oss-cn-hangzhou.aliyuncs.com/4.2/supos/196.jpg" />
- <strong>System Title</strong>: Set the title of supOS.

:::info
- The title is displayed on the browser tab when in the operation center and design center.
- <em>APP title-system title</em> is displayed as the page name of the low-code application.
- <em>Task status_task name (process name)-system title</em> is displayed as the name of task details page.
:::
- **Favicon**: Icon displays on the right side of the **System Title**.
- <strong>Navigation bar LOGO</strong>: Set the logo of supOS. The recommended and minimum image size is 110 Ã— 36.

<img width={450} src="https://wordpressfreezonex.oss-accelerate.aliyuncs.com/4.2/supos/229.png" />

- <strong>System Desktop Configuration</strong>: Select an background image for the default homepage that is configured in <a href="#setting-default-home-page">Setting Default Home Page</a>. 
- <strong>Login Page Configuration</strong>: Set the copyright, logo, background, slogan and login title (image) of the login page.

<img width={650} src="https://wordpressfreezonex.oss-accelerate.aliyuncs.com/platform/13.jpg" />

## Setting Login Time
1. Select <strong>System Configuration</strong> > <strong>System Settings</strong> > <b>Login time configuration</b>.

<img width={250} src="https://wordpressfreezonex.oss-cn-hangzhou.aliyuncs.com/4.2/supos/199.jpg" />

- <b>Login expire time</b>: When supOS on web browser or mobile clients
stays inactive for a period of time that exceeds the set duration,
you need to log in again.

:::info
- The login expiration settings take effect upon second login.
- You can set the platform to stay active on the APP setting page.
:::


- <b>Unique login</b>: You can only log in to supOS on one browser or start one session on supMobile at the same time when it is enabled.

:::info
- It is recommended to enable unique login to save online user quota.
- The unique login does not apply to admin.
:::

- <b>Relogin after close browser</b>: Click <b>yes</b> to enable request of relogin after closing browser.

:::info
When using Safari, close the browser by clicking the exit button.
:::

2. Click <strong>Save</strong>.

## Setting International Management

You can add UI strings and corresponding translations to sopOS to switch the platform language display.

:::info
Presently, supOS only supports simplified Chinese, traditional Chinese, English, Indonesian and German.
:::

1. Select <strong>System Configuration</strong> > <strong>System Settings</strong> > <strong>International Management</strong>, and then click <strong>Language Settings</strong>.

<img width={450} src="https://wordpressfreezonex.oss-cn-hangzhou.aliyuncs.com/4.2/supos/231.png"/>

2. Enable languages you need, and then click <strong>OK</strong>.

:::info
The enabled languages are displayed on the login page for selection.
:::
3. Click **New**, and then enter the UI string you want to add along with its location, key and translations for enabled languages.

<img width={450} src="https://wordpressfreezonex.oss-accelerate.aliyuncs.com/platform/57.png"/>

4. Click **OK** and the string is added.

:::tip
- Click <img src="https://wordpressfreezonex.oss-accelerate.aliyuncs.com/platform/23.png"/>, and use the template to batch import strings.
- Click <strong>Export</strong>, and then change the exported file and import again to update the current translation.
:::

## Configuring Log Policy

Various logs build up with the use of supOS, set a strategy to clean the log data for releasing storage space and improving the platform performance.

1. Select <strong>System Configuration</strong> > <strong>System Settings</strong> > <strong>Log policy configuration</strong>.
2. Set the strategy for each type of logs.

<ul>
<ul>
<li>The logs can be kept for up to 7300 days.</li>
<li>10 million pieces of logs can be saved at most.</li>
<li>Set the time for clearing log data separately to prevent server from overloading.
</li>
</ul>
</ul>

:::info
supOS keeps all logs permanently by default.
:::

<img width={650} src="https://wordpressfreezonex.oss-cn-hangzhou.aliyuncs.com/4.2/supos/200.jpg"/>

3. Click <strong>Save</strong>.

:::info
supOS clears the earliest log data when the set limit is reached.
:::

## Managing Time Series Data Storage
Set the storage duration of the time series data history. When the duration exceeds the set value, the platform deletes the earliest data and keep the latest.

1. Select <strong>System Configuration</strong> > <strong>System Settings</strong> > **Time series data storage policy**.
2. Set the duration, and then click **Save**.

:::info
- The default duration is one day. **0** means permanent storage.
- 7 GB of space is needed to store 10,000 points of data that changes by the second.
:::

<img width={650} src="https://wordpressfreezonex.oss-cn-hangzhou.aliyuncs.com/4.2/supos/234.png"/>

## Setting Allowlist and Blocklist

Set the allowlist and blocklist to control platform access. IPs on the blocklist cannot access supOS.

1. Select <strong>System Configuration</strong> > <strong>System Settings</strong> > <b>Allowlist &amp; Blocklist</b>.
2. Click the upper tab <strong>Allowlist</strong>, click <strong>New</strong>, and then enter IP to add new. Wildcard characters are available.

<img width={550} src="https://wordpressfreezonex.oss-accelerate.aliyuncs.com/4.2/supos/211.jpg" />

:::info
- Add the current IP to the list first. Otherwise it will be forced to log out.
- Blacklist works the similar way.
:::

:::note
Fuzzy match is available when searching for IPs.
:::

## Editing Cascading Setup
Connect the data of supOS A to the current supOS B.
:::tip what you need to do first
Add a GRPC connector under **Collection Node Management** > **Authentication** on supOS B, and obtain the node name and UUID.
:::

1. Select <strong>System Management</strong> > <strong>System Configuration</strong> > <strong>System Settings</strong> > <strong>Cascading Setup</strong> on supOS A.
2. Enter the node name, UUID and supOS B address.
3. Leave the port as default, and then click <strong>Save</strong>.

<img width={350} src="https://wordpressfreezonex.oss-cn-hangzhou.aliyuncs.com/4.2/supos/235.png"/>

4. Select **Collection Node Management** > **Status**, and authorize the connection.
5. Select **Object Model Management** > **Object Instance** > **Collector Template**, and check the instance generated from the connection.
