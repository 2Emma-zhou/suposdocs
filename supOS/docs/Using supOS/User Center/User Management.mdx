
Add users based on the post and assign corresponding permissions to achieve easy management.

### Adding User

1. Log in to supOS, and then click <img src="https://wordpressfreezonex.oss-accelerate.aliyuncs.com/platform/7.png"/> at the upper-right corner to go to the design center.
2. Select <strong>User Security Management</strong> > <strong>User Management</strong>, and then click <strong>Add</strong>.
3. Select a user type and then enter the information.

### Assigning Permission

Aside from the permissions of the role, you can assign extra
permissions to the user.

<ul>
<li>Admin can only assign permissions to users in the same company.</li>
<li>Branch admin can only assign APP permissions to user after the system admin assigns APP permission to the branch.</li>
</ul>

1. Log in to supOS, and then click <img src="https://wordpressfreezonex.oss-accelerate.aliyuncs.com/platform/7.png"/> at the upper-right corner to go to the design center.
2. Select <strong>User Security Management</strong> > <strong>User Management</strong>, and then click <strong>Permission</strong> under <strong>Operate</strong> of a user.

- **Function Permission**: Permissions of functions on supOS.

<ol>
<ol>
<li>Select <strong>Function Permission</strong>, and then click <b>User Permission</b>.

- **User Permission**: Permissions manually assigned to the user.
- **Inherited Permissions**: Permissions inherited from roles.

</li>
<li>Select a component, and then select functions inside the component whose access you want to grant to user.
<ul>
<li><strong>Operation Center</strong>: Includes permissions of default APPs (<strong>Desktop</strong> and <strong>Business DIY</strong> in <strong>Business Designer</strong>), third party APPs and designed industrial APPs.</li>
<li><strong>Design Center</strong>: Permissions to access menus of the design center.</li>
<li><b>Permission Assigned</b>: Permissions assigned to the user.</li>
</ul>
</li>
<li>Click <b>Assign Permission</b> at the upper-right corner to assign the selected function permissions to the current user.</li>
</ol>
</ol>

:::tip
Click <img src="https://wordpressfreezonex.oss-accelerate.aliyuncs.com/platform/26.png"/> next to <strong>Sync to user</strong>, you can select users in the company and assign the selected permissions to them. Up to 10 users and 50 permission can be assigned at the same time.
:::

<img src="https://wordpressfreezonex.oss-accelerate.aliyuncs.com/4.2/supos/205.png"/>

:::tip
You can select an assigned permission, and click the trash bin at the upper-right corner to delete it.
:::

- **Resource Permission**: Permissions to access resources of third-party apps after installation through API.
:::info
This tab can only be seen when resource permission is enabled under **System Configuration** > **System Settings** > **Resource permission config**.
:::

<ol>
<ol>
<li>Select <strong>Resource Permission</strong>, and then click <img src="https://wordpressfreezonex.oss-accelerate.aliyuncs.com/4.2/supos/206.png"/> to add a resource set.</li>
<li>Enter the resource set information, and then click <b>OK</b>. The callback address is the API supOS uses to access APP resources.
</li>
<li>Select the added resource, and then click <b>Assign Permission</b> at the upper-right corner.</li>
</ol>
</ol>

:::tip
Enable **Parent-child linkage**, when parent resources is selected, sub resources are selected by default.
:::

<img src="https://wordpressfreezonex.oss-accelerate.aliyuncs.com/4.2/supos/207.png"/>

- **Data Group**: Persmissions of object models.

<ol>
<ol>
<li>Click the <strong>Data Group</strong> tab.
<ul>
<li>
<strong>Resource Controlled</strong>: Assign data permissions based on the selected data group. It is the default option for all general users.
</li>
<li><strong>Resource Uncontrolled</strong>: Assign all data permissions. It is the default option for admin.
</li>
</ul>
</li>
<li>Select the data modeling permission, and then click <strong>Assign Permission</strong>.</li>
</ol>
</ol>
