For published services, you can apply access restrictions to ensure data security, and schedule health check on a regular basis.

### Applying Access Restrictions
1. Log in to supOS, and then select **Data Intelligence** > **Data Service** > **Service management**.
2. Select a service system, and then click **Access management**.
3. Enable **Access permission**, and then under **Whitelist**, click **New**.
4. On the pop-up window, click **New** again and enter the access key and IP address to which you want to grant permission.
![](https://wordpressfreezonex.oss-accelerate.aliyuncs.com/supDAM/60.png)
5. (optional) Under **Blacklist**, do the same to add IP addresses from which you want to block access.

### Scheduling Health check

1. On the **Service management** page, select a service system, and then click **Health check**.
2. Enable **Health check**, and then enter the service information to be checked.
- Health status return code: Set the return code and position.
- Start timed detection cycle: Set the health check schedule.
![](https://wordpressfreezonex.oss-accelerate.aliyuncs.com/supDAM/61.png)
3. Click **Save**, and then wait for at least one interval of the health check schedule, click **API list**.
![](https://wordpressfreezonex.oss-accelerate.aliyuncs.com/supDAM/62.png)
