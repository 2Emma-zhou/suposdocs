Once the data task is debugged successfully, you can set its dispatch schedule and publish it.
A data task can be triggered to run on scheduled time, upstream task or object instance event.

1. Click **Dispatch** on the top.
2. Under **Cycle dispatch**, set the cycle and specific time when the task is scheduled to run, and its expiry date.

![](https://wordpressfreezonex.oss-accelerate.aliyuncs.com/supETL/33.png)

3. Click **Message drive**, select the message source for triggering the task.
- Upstream task: Select an upstream task and set the task status when messages triggered.
![](https://wordpressfreezonex.oss-accelerate.aliyuncs.com/supETL/34.png)
- Object instance: Select an instance event on supOS to trigger the task.
![](https://wordpressfreezonex.oss-accelerate.aliyuncs.com/supETL/35.png)

4. Click **Confirm**.
5. Click **Publish** at the upper-right corner, and then you can select to directly publish the task or adjust it again before publish.

:::info
You can only change the task name and version, input and output data sources and dispatch schedule.
:::
