### Configuring on web

:::info
Make sure there is report developed on supOS.
:::

1. Log in to supLink web, and then click **App Manage**.
2. Under **Base App**, click **Edit** on **Notice**.

<img width={450} src="https://wordpressfreezonex.oss-accelerate.aliyuncs.com/supLink/41.png" />

<table>
    <thead>
      <tr>
        <th>Parameter</th>
        <th>Description</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td>QR code</td>
        <td>Use supLink app to scan the code to access the application directly.</td>
      </tr>
      <tr>
        <td>Visible range</td>
        <td>Set the application visible range. Select roles to grant permissions when set <b>Visible range</b> to <b>Part visible</b> or <b>Part invisible</b>.</td>
      </tr>
      <tr>
        <td>Select group</td>
        <td>Select an app group for the application.</td>
      </tr>
    </tbody>
</table>

3. Click **Back** after saving the configuration, and then click **Setting** on **Notice**.
4. Click **New**, and then enter the notice content.

<img width={450} src="https://wordpressfreezonex.oss-accelerate.aliyuncs.com/supLink/42.png" />

5. Click **Recipient**, select users from supOS and click **Publish**, and they will receive the notice on supLink mobile on the **Message** tab.

### Viewing Effect on Mobile

1. Install supLink mobile on your phone, start it and enter the supLink server address, tap **Start**.
2. Log in with the same account as supOS.
3. Tap **Message** on the bottom, and then view the notice.<br />

<img className="image-zoom" src="https://wordpressfreezonex.oss-accelerate.aliyuncs.com/supLink/43.png" />

4. Tap **Notice** under **Base App** to view all notices.
